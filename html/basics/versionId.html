The LDM Version Identifier

The LDM version identifier has the following form: 


major . minor . bug [.candidate]
where:
major
Is the major release-number. This number changes when the protocol between LDMs changes or when some other change occurs that renders the new version of the LDM incompatible with the previous version.
minor
Is the minor release-number. This number changes when the LDM acquires a new feature or capability or when this documentation must be modified to document a change (including a change to the instructions for building and installing the LDM).
bug
Is the bug-fix release-number. This number changes due to bug fixes in the LDM or performance improvements.
candidate
Is the release-candidate number. This field is for use by the UPC only and should not appear on any computer system outside that office's control.
